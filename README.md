## Требования к тестовому заданию

На Java и GO написать два микро-приложения с архитектурным стилем REST. Которые предоставляют клиентам по паре или более
методов из приведенного списка:

- электронная подпись стандарта PKCS#7 переданного документа (присоединенная или отсоединенная)
- проверка электронной подписи стандарта PKCS#7 переданного документа (присоединенная или отсоединенная)
- шифрование переданного клиентом сообщения
- дешифрование переданного сообщения
- расчет хэша переданного сообщения
- устанавливает защищенное доверенное соединение с каким-либо конкретным внешним ресурсом по HTTPS, загружает документ,
  отдает клиенту.

Интегрироваться с СУБД, исходные данные и результаты работы методов брать и соответственно записывать в БД, например H2
DataBase.

Для Java cреда и инструменты:

- Java 11+
- В качестве сервера, контейнера сервлетов Apache Tomcat/ Eclipse Jetty
- криптографические провайдеры Bouncy Castle / SUN / КриптоПро JCP

Алгоритмы:
Подпись RSA
Шифрование RSA (AES +RSA)
Хэш функция SHA-256 / PBKDF2

Контейнер обмена с приложением JSON

Генерация RSA ключей и сертификатов
Portecle /XCA /Open SSL/ прочие утилиты
Контейнеры с ключами JKS / P12

Сборка приложения Maven/Gradle

Плюсом будет

- настройка для приложения TLS/SSL и доступ к методам по HTTPS с дополнительной аутентификацией (SSL сертификат и ключи
  так же самостоятельно сгенерированные).
- JUnit тесты с вызовами реализованных методов.
- предусмотренная защита паролей к контейнерам с ключами, как минимум, не хранятся в коде в открытом виде.
- написание пользовательского интерфейса для приложения, реализующего тестовое задание front end WEB UI вызывающего
  сервисы back end с криптографическими функциями. Например, возможность загрузить файл и выполнить его подписание или
  шифрование
